Recursion

Posted on Feb 2, 2021

Hoje eu estava terminando a primeira parte (Basic Javascript) do Java Script Algorithms and Data Structures Certification do freeCodeCamp, especificamente o último desafio, quando aprendi algo importante (eu acho).

O exercício era o seguinte: como escrever uma função que, recebendo dois números, a saber, startNum e endNum, retorna um vetor com todos os números inteiros entre eles? O grande lance é que você não pode utilizar de expedientes como loops clássicos como as construções for e while. A saída é, portanto, utilizar funções recursivas.

Uma função recursiva nada mais é que uma função que invoca a si mesmo quando é executada. O exemplo clássico da matemática é o fatorial.

Fatorial(n) = n * Fatorial (n-1) = n * (n-1) * Fatorial (n-2)…

Funções recursivas precisam sempre (eu acho) de definições nos casos-limite. No caso do fatorial, temos que Fatorial(0) = 1. Mas isso eu já sabia e não foi o que chamou a minha atenção. Calma, eu vou chegar lá.

Abaixo está o código em javascript que utilizei para resolver o desafio.

function rangeOfNumbers(startNum, endNum) {
  if (startNum === endNum) {
    return [startNum];
  }
  else {
    var numbers = rangeOfNumbers(startNum, endNum - 1);
    numbers.push(endNum);
    return numbers;
  };
};

A parte dentro do else é a parte recursiva onde a função chama a si mesma reiteradamente até caso-limite. O caso limite do exercício estipula que quando startNum e endNum são iguais, a função deve retornar um array com o valor único. Foi isso que chamou minha atenção.

Na parte recursiva, a função chama a si esma com um parâmetro endNum - 1 e adiciona o valor de endNum ao final do array através do método .push(). Eu achei que essa formulação não ia funcionar porque a parte não-recursiva, contém return [startNum] e achei que isso ia encerrar a execução da função. Mas isso não acontece. Uma vez que a parte recursiva já está sendo executada, o return da parte não-recursiva não encerra a ação, mas apenas é repassado ao estágio “anterior” (no caso, a última chamada de rangeOfNumbers(startNum, endNum - 1).
